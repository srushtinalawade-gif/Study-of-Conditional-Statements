## Theory
In Python, conditional statements serve as the primary mechanism for non-linear execution. Instead of running every line of code sequentially, the interpreter evaluates specific expressions to determine which block of code satisfies the current criteria. This "branching" is what allows software to behave intelligently based on varying user input or system states.

Core Principles of Branching
Evaluation of Truthiness: Every condition is resolved to a Boolean value (True or False).

Whitespace Dependency: Python relies on strict indentation rather than curly braces to denote which lines belong to a specific branch.

Sequential Logic: In a ladder structure, the interpreter stops at the first condition that evaluates to True, ignoring all subsequent elif or else blocks.

Logical Composition: Multiple criteria can be linked within a single statement using operators to create sophisticated filters.

Functional Variations
Standard if: A solitary gatekeeper that runs code only when a specific requirement is met.

Binary if-else: A dual-path structure ensuring that one of two possible outcomes is always executed.

The elif Chain: A multi-path ladder used when there are more than two distinct categories or outcomes.

Hierarchical (Nested) if: Placing conditions inside existing blocks to create a "decision tree" for granular filtering.
